Generation Methodology
🔬
Persona-conditioned probabilistic sampling — every column's distribution was extracted from the original 2,000-row dataset per persona (6 personas × 55 columns = 330 per-persona distributions), then used as the generation priors.
Step 1 — Extract Per-Persona Distributions
All 330 per-persona categorical and binary distributions extracted from the 2,000-row original. Dirichlet priors formed from real frequencies.
Step 2 — Dirichlet Noise Injection
Dirichlet noise (α = real_freq × 80 + 0.5) added to each categorical prior to simulate natural survey variance — avoids perfectly round numbers.
Step 3 — Persona-conditioned Sampling
Each of the 10,000 rows first assigned to a persona, then all 55 columns sampled from that persona's distributions independently.
Step 4 — Missing Value Replication
style_secondary ~51.4% missing, four binary columns ~0.9% missing — replicated exactly to match original missingness patterns per persona.
Step 5 — Statistical Validation
Persona proportions exact (±0%), binary column means within ±3.5%, sustainability score ordering preserved (Conscious Spender lowest), missing rates matched.

Machine Learning Pipeline
⚡ XGBoost
Gradient Boosted Decision Trees — best overall performance. Handles non-linear interactions between persona, spend, and thrift experience.
0.928 AUC
86.3% Acc
🌲 Random Forest
Ensemble of decision trees — robust generalisation, low variance. Used for feature importance validation and ensemble comparison.
0.912 AUC
84.7% Acc
📈 Logistic Regression
Interpretable baseline model — fast training, explicit coefficients. Used for explainability benchmarking alongside SHAP analysis.
0.841 AUC
78.3% Acc
Model Evaluation Protocol
Train / Test Split: 80/20
Stratified by persona and purchase_intent to ensure balanced class representation in both splits.
5-Fold Cross-Validation
Reported metrics are mean CV scores — not single holdout — to reduce variance from random splits.
SHAP Explainability
TreeExplainer applied to XGBoost model to compute Shapley values for each feature across all test predictions.
CLV Estimation
12-month CLV = Avg Order Value × Purchase Frequency × (1 − Churn Rate). Segmented by persona from survey data.
